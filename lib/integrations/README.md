# Future integration skeletons

These files are intentionally inert placeholders (build spec §7). Each exports a
clearly-marked stub so the current build never breaks. Wire them up when the
corresponding feature is needed.

| File             | Purpose                          |
| ---------------- | -------------------------------- |
| `cms.ts`         | Headless CMS content source      |
| `auth.ts`        | Authentication (e.g. Auth0)      |
| `booking.ts`     | Booking / scheduling system      |
| `ecommerce.ts`   | E-commerce / checkout            |
